# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this repository, please report it by opening a GitHub Issue using the **Security Vulnerability Report** template.

Do not include sensitive details (live credentials, exploit payloads) in the issue body. If the finding is highly sensitive, contact the security team directly at kalp.modi@devxlabs.ai before filing.

## What to Include

Use the issue template — it will prompt you for:
- Vulnerability type and severity
- Affected file and line number
- Steps to reproduce or proof of concept
- Suggested fix (if known)

## Response SLA

| Severity | First Response | Target Fix |
| -------- | -------------- | ---------- |
| Critical | 24 hours       | 3 days     |
| High     | 48 hours       | 7 days     |
| Medium   | 5 days         | 30 days    |
| Low      | 14 days        | Next release |

## Supported Versions

All active branches under `devx-commerce` are in scope. Archived or deprecated repos are out of scope.
